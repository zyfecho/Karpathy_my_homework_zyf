## Q

SPI和I2C有什么区别


## A

SPI 和 I2C 的主要区别如下：

| 对比项 | SPI | I2C |
|---|---|---|
| 通信方式 | **同步、全双工** | **同步、半双工** |
| 线数 | **至少4根**：MOSI、MISO、SCL、CS | **2根**：SDA、SCL |
| 速度 | **更快** | 相对较低 |
| 寻址方式 | 通过 **CS片选** 选择从机 | 通过 **设备地址** 选择从机 |
| 应答机制 | **没有 ACK 机制** | **有 ACK 应答机制** |
| 多设备连接 | 每个从机通常需要单独一根 CS | 支持 **一主多从**，共用总线 |
| 硬件特点 | 线多、结构直接、吞吐量高 | 线少、布线简单，但协议更复杂一些 |

补充说明：

- **SPI** 适合对速度要求高的场景，因为它没有地址开销和 ACK 机制，通信效率更高。(来源: spi.md)
- **I2C** 适合引脚资源紧张、需要挂多个从设备的场景，因为只需要两根线，并且支持从机地址寻址。(来源: i2c.md)
- I2C 的引脚通常是 **开漏输出 + 上拉电阻**，而 SPI 没有这个典型要求。(来源: i2c.md)
- 面试里常见总结是：**SPI速度快、全双工；I2C线少、支持多从机寻址、有ACK确认。** (来源: interview-guide.md)

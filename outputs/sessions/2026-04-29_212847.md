## Q

Freertos优先级设置方法


## A

在知识库里，FreeRTOS 优先级设置主要有两种方式：

## 1. 通过代码动态设置
使用 CMSIS v2 封装函数：

```c
osThreadSetPriority(任务句柄, 优先级);
```

对应的 FreeRTOS 原版函数是：

```c
vTaskPrioritySet(任务句柄, 优先级);
```

它用于**更改任务优先级**。(来源: freertos-task-management.md)

同时，也可以获取任务当前优先级：

```c
osThreadGetPriority(任务句柄);
```

对应原版：

```c
uxTaskPriorityGet(任务句柄);
```

(来源: freertos-task-management.md)

---

## 2. 通过 CubeMX / 硬件配置时设置
知识库提到：

- **优先级范围：硬件设置（CubeMX）时为 0~31**
- 软件设置时可自定义

也就是说，在 CubeMX 创建 RTOS 任务时，可以直接给任务配置优先级。(来源: freertos-task-management.md)

---

## 补充说明
- 每个时刻只执行一个任务
- **高优先级任务优先执行**
- 新任务创建后会放入就绪列表，由调度器决定运行哪个任务 (来源: freertos-task-management.md, freertos-basics.md)

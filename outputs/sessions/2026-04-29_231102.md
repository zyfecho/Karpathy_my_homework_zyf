## Q

SPI和I2C有什么区别


## A

SPI 和 I2C 的主要区别如下：

| 对比项 | SPI | I2C |
|---|---|---|
| 通信方式 | **全双工** | **半双工** |
| 同步方式 | **同步** | **同步** |
| 线数 | 至少 **4 根**：MOSI、MISO、SCL、CS | **2 根**：SDA、SCL |
| 速度 | **更快** | **较低** |
| 设备选择方式 | 通过 **CS/SS 片选线** 选择从机，每个从机通常一根 CS | 通过 **地址寻址** 选择从机，一主多从 |
| 应答机制 | **没有 ACK 机制** | **有 ACK 应答机制** |
| 硬件特性 | 结构简单，吞吐量高 | 需要 **开漏输出 + 上拉电阻** |
| 适合场景 | 高速通信 | 引脚紧张、多从机场景 |

补充几点：

- **SPI** 使用 `MOSI / MISO / SCLK / CS`，主机产生时钟，通信时可以同时发送和接收数据。(来源: spi.md)
- **I2C** 使用 `SDA / SCL` 两根线，支持 **起始信号、停止信号、ACK应答、重复起始** 等协议机制。(来源: i2c.md)
- 面试里常见总结是：**SPI 更快、全双工；I2C 线更少、支持多从机寻址、带 ACK。** (来源: interview-guide.md)

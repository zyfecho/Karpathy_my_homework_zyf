## Q

FREERTOS优先级规则？


## A

根据知识库，FreeRTOS 的优先级规则主要有这些：

1. **高优先级任务优先执行**  
   多任务系统中，**优先级高的任务会优先获得CPU**，因此实时性更好。(来源: freertos-basics.md)

2. **每个时刻只执行一个任务**  
   调度器在某一时刻只会运行一个任务，通常从“就绪态”任务中选择优先级最高的那个执行。(来源: freertos-task-management.md)

3. **新创建的任务进入就绪列表**  
   任务创建后不会立刻“并行运行”，而是先加入活动/就绪任务列表，等待调度。(来源: freertos-task-management.md)

4. **优先级范围**  
   知识库中提到：  
   - 在 **CubeMX 硬件设置** 时，优先级范围是 **0~31**  
   - 软件设置时可自定义范围 (来源: freertos-task-management.md)

5. **恢复高优先级任务会立即抢占**  
   如果一个被挂起的任务通过 `osThreadResume` 恢复后，**其优先级高于当前任务**，系统会**立即切换**到该任务执行。(来源: freertos-task-management.md)

## Q

STM32中UART的中断和DMA模式区别


## A

STM32 中 UART 的**中断模式**和 **DMA 模式**，知识库里主要能确认的是：它们都是 HAL 库支持的非阻塞通信方式，但使用机制不同。可总结如下：

## 1. HAL接口不同

### 中断模式
```c
HAL_UART_Transmit_IT(&huart, pData, Size);
HAL_UART_Receive_IT(&huart, pData, Size);
```

### DMA模式
```c
HAL_UART_Transmit_DMA(&huart, pData, Size);
HAL_UART_Receive_DMA(&huart, pData, Size);
```
(来源: uart.md), (来源: hal-quick-reference.md)

---

## 2. 回调方式

UART 常见回调函数包括：

```c
HAL_UART_TxCpltCallback(&huart);          // 发送完成
HAL_UART_RxCpltCallback(&huart);          // 接收完成
HAL_UARTEx_RxEventCallback(&huart, Size); // 接收事件（如IDLE触发）
```
这些回调通常用于在中断或接收事件完成后处理数据。(来源: uart.md), (来源: hal-quick-reference.md)

---

## 3. 两者本质区别

知识库明确给出了 UART 有三种工作方式：**阻塞（Polling）**、**中断（IT）** 和 **DMA**。(来源: hal-quick-reference.md)

但对于“中断模式和 DMA 模式的底层差异、CPU占用、适用场景”的详细展开，知识库中**没有进一步说明**。  
也就是说，我只能基于资料明确回答：

- **中断模式**：通过 `HAL_UART_Transmit_IT` / `HAL_UART_Receive_IT` 使用  
- **DMA模式**：通过 `HAL_UART_Transmit_DMA` / `HAL_UART_Receive_DMA` 使用  
- 两者都不是阻塞方式，都可配合完成回调函数使用 (来源: uart.md), (来源: hal-quick-reference.md)

---

## 4. 如果按面试简答来写

可以写成：

- **中断模式**：UART 在发送/接收过程中通过中断通知CPU处理数据，调用 `_IT` 接口  
- **DMA模式**：UART 通过 DMA 搬运数据，调用 `_DMA` 接口  
- 两者都比阻塞方式更适合异步通信场景 (来源: uart.md), (来源: hal-quick-reference.md)

## Q

GPIO的几种模式？


## A

知识库里这部分主要讲了 **GPIO 输入输出中的输出模式**，明确提到两种：

## 1. 推挽输出（Push-Pull）
- 具有强驱动能力
- 可以**主动输出高电平和低电平**
- **一般不需要配置上下拉电阻**
- 如果再配上拉/下拉，可能会浪费电流，还可能影响电平变化速度 (来源: gpio.md)

## 2. 开漏输出（Open-Drain）
- 只能**主动拉低**
- 输出高电平要依赖**上拉电阻**
- 因此通常要配置为：开漏输出 + 上拉
- 常见应用是**I2C通信**、电平转换 (来源: gpio.md), (来源: i2c.md)

---

## 知识库里还补充了几点
### 输出模式下需要上下拉的特殊场景
1. 开漏输出模式必须配上拉  
2. 对系统安全性要求高时，可在上电未初始化阶段通过上下拉防止误动作  
3. 某些外部设备要求 MCU 未初始化时引脚就保持确定电平 (来源: gpio.md)

### 常用HAL函数
```c
HAL_GPIO_WritePin(端口, 引脚号, 电平状态);
HAL_GPIO_ReadPin(端口, 引脚号);
HAL_GPIO_TogglePin(端口, 引脚号);
```
(来源: gpio.md), (来源: hal-quick-reference.md)
